Built from Scratch. Engineered to Perform.

Enginx is a high-performance HTTP/1.1 web server written entirely in C++ & C — featuring non-blocking I/O, NGINX-style config parsing, CGI execution, session management, file uploads, themed error pages, and dark/light theming.

Core Capabilities

Everything a Web Server Needs

From raw TCP sockets to CGI scripts and cookie-based sessions — Enginx handles it all.

🔀

I/O Multiplexing

Handles thousands of concurrent connections through poll()-based event-driven architecture with non-blocking sockets.

📝

NGINX-Style Config

Full compiler pipeline: lexical tokenizer → abstract syntax tree → semantic validation. Server blocks, locations, MIME types.

⚙️

CGI Execution

Run Python & C# scripts via Common Gateway Interface with environment variables, pipe IPC, and timeout management.

🍪

Session Management

Cookie-based session system with Set-Cookie headers, SESSIONID tracking, persistent theme preferences across pages.

📤

File Upload & Deletion

Upload files via POST with configurable body size limits (1MB, 64B). Delete uploaded files with DELETE method support.

🗂️

Auto-Index Directory

Beautiful file manager UI with toggle-able directory listing. Autoindex on/off per location block.

🔄

HTTP Redirections

301 permanent & 302 temporary redirects configured per location. External redirect support (e.g., to Google).

🏠

Virtual Hosting

Multiple server blocks on different ports (1025, 1026) with independent roots, indexes, and location rules.

🎨

Dark / Light Theme

Cookie-persisted theme toggle on every page — index, error pages, autoindex, and CGI scripts all respect the theme.

📄

MIME Type System

Custom types block mapping 8+ MIME types: HTML, CSS, JS, JPEG, PNG, GIF, plain text, and binary octet-stream.

🗺️

Smart Routing

Location matching with root remapping, per-location method restrictions, index file resolution, and path normalization.

📋

Structured Logging

Complete request/response logging to webserver.logs with timestamps, status codes, and connection tracking.

Interactive

Configuration

NGINX-Style Config Engine

A full compiler pipeline: tokenizer → AST → semantic validator.

Production-Ready Parsing

Enginx parses configuration files using the same paradigm as real-world compilers — lexical tokenization, abstract syntax tree generation, and semantic validation.

  • ✓ Multiple server blocks with virtual hosting
  • ✓ Location directives with root remapping
  • ✓ CGI pass for .py and .cs extensions
  • ✓ Autoindex on/off per location
  • ✓ Client body size limits (1m, 64b)
  • ✓ 301/302 redirections per location
  • ✓ Per-location method restrictions
  • ✓ Custom error pages per server
  • ✓ File upload & delete toggles
  • ✓ Custom MIME type definitions
engineX.conf
1server { 2 listen 1025; 3 server_name localhost; 4 root HTTP/DefaultPages; 5 autoindex off; 6 client_max_body_size 1m; 7 error_page 404 /errors/404.html; 8 9 location /scripts { 10 cgi_pass .py /usr/bin/python3; 11 } 12 location /upload { 13 client_body_in_file_only on; 14 delete_files on; 15 } 16 location /google { 17 return 301 https://www.google.com; 18 } 19}
